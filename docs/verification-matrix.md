## Gaps

- **§7 SSE end-to-end.** A full `/stream` integration test exists but is `pytest.skip`-marked because `httpx.ASGITransport` hangs the pytest worker on streaming responses. The bus, the publisher (state machine), the dispatcher (ntfy), and the multiplexer (`/stream`) each have unit coverage; what's missing is one end-to-end assertion that bytes hit the wire. Manual: `curl -N http://127.0.0.1:8765/stream` while triggering a hook in another terminal.
- **§8 #2 / #3 / #9 (manual install + 2 s SLA + PWA on iPhone).** These need a real Mac + iPhone. Documented manual steps for the release-day checklist.
- **§8 #8 + §10 #6 (collector restart + launchd KeepAlive).** Persistence is unit-tested; the `launchctl kickstart` → `csm doctor` chain requires a live Mac.
- **§10 #4 (Keychain MDM lockout) + §11 (Husky enforcement).** Behavioural — not feasible to unit-test the OS-level paths or the git hook itself. Sanity-check on a real Mac.

No remaining ❌ rows: the CLI's `csm install-hooks` and `csm doctor` subcommands have landed, closing the two pre-v0.1.0 gaps the original matrix identified.

## How to re-run

When preparing v0.1.0 (or any subsequent release):

1. **Run the automated suite.**
   ```bash
   make test
   ```
   This invokes pytest under `packages/collector` and Vitest under `packages/dashboard`. Every `✅` row above should be green; any new failure is a release blocker.

2. **Walk the manual rows.** For every `⚠️` or `❌` row, perform the documented procedure on a real Mac and tick it off in a release-day checklist. Capture the gate test outcome in `docs/gate-result.md`.

3. **Refresh the matrix.** When new MUSTs land in the spec (or the spec is amended), add or update rows here. Cap is 25–40 rows; if you exceed the cap, group related MUSTs onto a single row that points at multiple test paths.

4. **Tag the release.** Once the matrix is fully `✅` (modulo documented manual rows) and all `❌` rows are addressed, tag `v0.1.0` per the procedure in [`HANDOFF.md`](HANDOFF.md).
